SOCIALHUB - START HERE
=======================

Welcome to SocialHub! This is a complete, production-ready monorepo for managing
multiple social media platforms from a single dashboard.

WHAT IS SOCIALHUB?
==================
SocialHub is a multi-platform social media management tool that allows users to:
- Manage Facebook, Instagram, YouTube, TikTok, and LinkedIn accounts
- Create and schedule posts across multiple platforms
- Track analytics and engagement metrics
- Upload and organize media files
- Connect multiple brands with separate social accounts

QUICK START (5 MINUTES)
======================

1. Install Dependencies
   $ npm install

2. Set Up Environment
   Create .env file in root with database connection:
   DATABASE_URL=postgresql://localhost/socialhub

3. Initialize Database
   $ npm run db:setup

4. Start Development Servers
   $ npm run dev
   Backend: http://localhost:3000
   Frontend: http://localhost:5173

5. Login to Dashboard
   - Register new account at /register
   - Login at /login
   - Create a brand
   - Connect social accounts via OAuth

API ENDPOINTS (FOR DEVELOPERS)
==============================

Authentication:
  POST   /auth/register       Register new user
  POST   /auth/login          User login
  GET    /auth/me             Current user

Brands:
  GET    /brands              List all brands
  POST   /brands              Create brand
  GET    /brands/:id          Get brand details

Posts:
  GET    /posts               List posts
  POST   /posts               Create post
  PUT    /posts/:id           Update post
  POST   /posts/:id/publish   Publish immediately
  POST   /posts/:id/schedule  Schedule for later

Media:
  GET    /media               List media
  POST   /media/upload        Upload file
  DELETE /media/:id           Delete file

OAuth:
  GET    /oauth/authorize/:platform     Start OAuth flow
  POST   /oauth/callback/:platform      Handle callback


DATABASE MODELS
===============

User
  - Email & password (encrypted)
  - Profile info (name, avatar)
  - Created/updated timestamps

Brand
  - Name & description
  - Logo & website
  - Belongs to one user

OAuthAccount
  - Connected social account
  - Platform-specific tokens
  - Linked to brand & user

Post
  - Content & title
  - Scheduled/published dates
  - Status: draft/scheduled/published/failed
  - Attached media files

Media
  - File storage key
  - Type: image/video/document
  - Dimensions & size

Analytics
  - Daily metrics per platform
  - Views, engagement, comments
  - Linked to brand

TROUBLESHOOTING
===============

Issue: "Cannot connect to database"
Solution: Check DATABASE_URL in .env and ensure PostgreSQL is running
  $ psql -U postgres (test connection)

Issue: "Redis connection error"
Solution: Start Redis service
  $ redis-server  (or sudo systemctl start redis-server)

Issue: Port 3000 already in use
Solution: Kill existing process or change PORT in .env
  $ lsof -i :3000
  $ kill -9 <PID>

Issue: Dependencies installation fails
Solution: Clear cache and retry
  $ npm cache clean --force
  $ rm -rf node_modules
  $ npm install
